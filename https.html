HTTP and HTTPS Protocols

HTTPS stands for Hyper Text Transport Protocol Secure

HTTP is one of the main protocols at the application layer that allows computers to communicated with each other. Think of it as the language that one computer uses to ask another computer for a document. HTTP works by sending HTTP requests to a server. There are two types of HTTP requests: GET requests and POST requests. GET requests are much more common as they are mostly used for requesting web pages. For example, GET requests are used for getting images and HTML files. HTTP POST requests are for when you have to send a information like filling up a form or typing up a search query. For example, when you log in to Tumblr. the first thing you do is make a POST request. This POST request has some data attached to it. It has your email address and password. This POST requests then goes to Tumblr's server where it will check with its database for a match. If successful, it sends a web page back to your browser allowing access to your Tumbler account.

HTTPS is an extension of HTTPS and functions just like HTTP but is designed to be more secure and offers more protection of your data. It does this by by asking your web browser to communicate on a secure channel using something called Secure Sockets Layer and its successor Transport Layer Security.
